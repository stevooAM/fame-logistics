  Run the following grep/inspection checks against the `backend/` tree. Record the exact command, the result (count and representative lines), and a PASS / FAIL / REVIEW verdict for each item. Do NOT modify any source files — this is a read-only audit.

  Check categories (document findings under matching SEC-0x headings):

  1) SEC-01 (HTTPS) — Confirm plan 10-01 added SECURE_SSL_REDIRECT, HSTS, SECURE_PROXY_SSL_HEADER. If plan 10-01 has not run yet, mark this item as "PENDING (awaiting 10-01)".

  2) SEC-02 (bcrypt 12 rounds) — grep `BCryptSHA256PasswordHasher` in settings.py. Also run `python manage.py shell -c "from django.contrib.auth.hashers import get_hashers; h = [x for x in get_hashers() if 'BCrypt' in x.__class__.__name__][0]; print(h.rounds)"` (should be 12). If the shell is unavailable in the execution environment, document the config-level check only.

  3) SEC-03 (CSRF) — grep for `CsrfViewMiddleware` in settings. Grep every ViewSet for `@csrf_exempt` usage (must be 0, except JWT login which uses SessionAuthentication). Command: `grep -r "csrf_exempt" backend/ --include="*.py"`

  4) SEC-04 (no raw SQL) — grep for raw SQL outside migrations:
     - `grep -rn "\.raw(" backend/ --include="*.py" | grep -v migrations/` (expect 0)
     - `grep -rn "cursor\.execute" backend/ --include="*.py" | grep -v migrations/` (expect 0)
     - `grep -rn "RunSQL" backend/ --include="*.py" | grep -v migrations/` (expect 0)

  5) SEC-05 (XSS) — grep the frontend for dangerous sinks applied to user data:
     - `grep -rn "dangerouslySetInnerHTML" frontend/src/` — list every hit and confirm the value is either a static constant or a library-generated string (e.g., a chart SVG), NEVER raw user input.
     - `grep -rn "innerHTML" frontend/src/` (expect 0 outside test fixtures).
     - For any hit, inspect the file and record who controls the source string.

  6) SEC-06 (rate limit) — confirm `LoginRateLimitMiddleware` is in settings MIDDLEWARE and `LOGIN_RATE_LIMIT_MAX == 10`, `LOGIN_RATE_LIMIT_WINDOW == 900`. Cite `backend/core/middleware.py` line numbers.

  7) SEC-07 (cookies) — confirm `CookieJWTAuthentication` sets HttpOnly + Secure + SameSite. grep the login view for `samesite=`, `secure=`, `httponly=` kwargs. Also record the production-only cookie flags introduced by plan 10-01.

  8) SEC-08 (audit log) — grep for `AuditLogMixin` and list every ViewSet that inherits it:
     - `grep -rn "AuditLogMixin" backend/ --include="*.py"`
     - Cross-reference against `grep -rn "class .*ViewSet" backend/ --include="*.py"` to spot any ViewSet that does NOT inherit the mixin. Record the list. Any non-read-only ViewSet missing the mixin is a BLOCKER.

  9) SEC-09 (server-side validation) — spot-check 5 DRF serializers (customers, jobs, approvals, accounts/invoices, setup lookups): open each, confirm a `*Serializer(ModelSerializer)` or `Serializer` exists and lists explicit `fields` / `validators`. Record file paths and line numbers.

  10) PERF-02 (server-side pagination) — grep for `PageNumberPagination` and the global `PAGE_SIZE` setting. List any ViewSet that overrides `pagination_class = None` (BLOCKER if present on a listing endpoint that returns grid data).

  Anti-patterns to specifically scan for:
  - `DEBUG = True` hardcoded: `grep -rn "DEBUG = True" backend/ --include="*.py" | grep -v migrations/`
