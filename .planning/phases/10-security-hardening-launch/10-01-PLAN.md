  <name>Task 1: Harden Django settings for production</name>
  <files>backend/config/settings.py</files>
  <action>
  Update `backend/config/settings.py` with production-safe configuration. Keep all existing logic for DEBUG=True unchanged so local dev still works.

  1. Change the DEBUG default to False:
     ```python
     DEBUG = os.environ.get("DJANGO_DEBUG", "False") == "True"
     ```

  2. Change ALLOWED_HOSTS default to an empty list (must be set via env in prod):
     ```python
     ALLOWED_HOSTS = [h.strip() for h in os.environ.get("DJANGO_ALLOWED_HOSTS", "").split(",") if h.strip()]
     if DEBUG and not ALLOWED_HOSTS:
         ALLOWED_HOSTS = ["localhost", "127.0.0.1", "backend"]
     ```

  3. Add CSRF_TRUSTED_ORIGINS from env (required for cross-origin POST over HTTPS):
     ```python
     CSRF_TRUSTED_ORIGINS = [
         o.strip() for o in os.environ.get("CSRF_TRUSTED_ORIGINS", "").split(",") if o.strip()
     ]
     ```

  4. Add a guarded production security block at the end of the file (after CORS, before Cloud Storage):
     ```python
     # ------------------------------------------------------------------
     # Production security hardening (only active when DEBUG=False)
     # ------------------------------------------------------------------
     if not DEBUG:
         # HTTPS enforcement
         SECURE_PROXY_SSL_HEADER = ("HTTP_X_FORWARDED_PROTO", "https")
         SECURE_SSL_REDIRECT = True

         # HSTS — 1 year, include subdomains, eligible for preload
         SECURE_HSTS_SECONDS = 31_536_000
         SECURE_HSTS_INCLUDE_SUBDOMAINS = True
         SECURE_HSTS_PRELOAD = True

         # Browser hardening headers
         SECURE_CONTENT_TYPE_NOSNIFF = True
         SECURE_REFERRER_POLICY = "same-origin"
         X_FRAME_OPTIONS = "DENY"

         # Cookie flags (CookieJWTAuthentication already writes Secure+HttpOnly, but
         # these cover Django's session & CSRF cookies as well)
         SESSION_COOKIE_SECURE = True
         SESSION_COOKIE_HTTPONLY = True
         SESSION_COOKIE_SAMESITE = "Lax"
         CSRF_COOKIE_SECURE = True
         CSRF_COOKIE_HTTPONLY = False  # must be readable by frontend for DRF SessionAuth
         CSRF_COOKIE_SAMESITE = "Lax"

         # Fail closed — refuse to boot in prod with the insecure default SECRET_KEY
         if SECRET_KEY.startswith("django-insecure-"):
             raise RuntimeError(
                 "DJANGO_SECRET_KEY is not set in production. Refusing to start."
             )
     ```

  5. Replace the DRF block so the Browsable API renderer is stripped in prod:
     ```python
     REST_FRAMEWORK = {
         "DEFAULT_AUTHENTICATION_CLASSES": (
             "core.authentication.CookieJWTAuthentication",
         ),
         "DEFAULT_PERMISSION_CLASSES": ("rest_framework.permissions.IsAuthenticated",),
         "DEFAULT_PAGINATION_CLASS": "rest_framework.pagination.PageNumberPagination",
         "PAGE_SIZE": 20,
         "DEFAULT_RENDERER_CLASSES": (
             ("rest_framework.renderers.JSONRenderer",)
             if not DEBUG
             else (
                 "rest_framework.renderers.JSONRenderer",
                 "rest_framework.renderers.BrowsableAPIRenderer",
             )
         ),
     }
     ```

  6. Tighten CORS: when DEBUG=False the existing `CORS_ALLOWED_ORIGINS` env var path already runs — verify no wildcards are introduced. DO NOT change the DEBUG branch (localhost regex). Add one guard right after the CORS block:
     ```python
     if not DEBUG and not CORS_ALLOWED_ORIGINS:
         raise RuntimeError("CORS_ALLOWED_ORIGINS must be set in production.")
     ```

  7. Add a DJANGO_ADMIN_ENABLED flag (defaults to False) near the top of the file, after ALLOWED_HOSTS:
     ```python
     DJANGO_ADMIN_ENABLED = os.environ.get("DJANGO_ADMIN_ENABLED", "True" if DEBUG else "False") == "True"
     ```

  Do NOT touch PASSWORD_HASHERS (bcrypt is already primary from Phase 2).
  Do NOT touch SIMPLE_JWT (access/refresh lifetimes are already correct).
  Do NOT rename or remove existing middleware/apps.
  </action>
