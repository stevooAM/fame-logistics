<task type="auto">
  <name>Task 1: Idle timeout hook with activity tracking</name>
  <files>frontend/src/hooks/use-idle-timeout.ts</files>
  <action>
    Create `frontend/src/hooks/use-idle-timeout.ts`:

    `useIdleTimeout` hook:
    - **Parameters**: `{ timeoutMs?: number, warningMs?: number, onTimeout: () => void, onWarning: () => void, onActivity?: () => void }`
    - **Defaults**: timeoutMs = 30 * 60 * 1000 (30 min), warningMs = 2 * 60 * 1000 (2 min before timeout = at 28 min mark)

    - **Activity tracking**:
      - Listen for: mousemove, mousedown, keydown, scroll, touchstart events on document
      - Throttle activity detection to once per 30 seconds (avoid excessive timer resets)
      - On activity: reset both warning and timeout timers

    - **Timer logic**:
      - `warningTimer`: fires at (timeoutMs - warningMs) milliseconds of inactivity -> calls onWarning
      - `timeoutTimer`: fires at timeoutMs milliseconds of inactivity -> calls onTimeout
      - Both timers reset on any detected activity

    - **Return value**: `{ resetTimers: () => void, isWarning: boolean }`
      - `resetTimers()` — called when user clicks "Stay logged in" to manually reset
      - `isWarning` — boolean state, true when warning timer has fired

    - **Cleanup**: Remove all event listeners and clear timers on unmount

    - **Implementation notes**:
      - Use useRef for timer IDs to avoid stale closure issues
      - Use useCallback for the throttled activity handler
      - Do NOT track activity on the login page (hook should only be mounted inside dashboard layout)
  </action>
  <verify>

    - `grep "mousemove\|keydown\|scroll" frontend/src/hooks/use-idle-timeout.ts` confirms activity events
    - `grep "warningTimer\|timeoutTimer" frontend/src/hooks/use-idle-timeout.ts` confirms dual timer logic
    - `grep "30.*60.*1000\|1800000" frontend/src/hooks/use-idle-timeout.ts` confirms 30 min default
  </verify>
