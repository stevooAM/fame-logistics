<task type="auto">
  <name>Task 1: User management serializers</name>
  <files>backend/core/serializers.py</files>
  <action>
Add the following serializers to the existing core/serializers.py:

1. **UserListSerializer** — for GET /api/users/ list response:
   - Fields: id, username, email, first_name, last_name, role (nested {id, name} using existing RoleSerializer), phone, department, is_active, is_force_password_change, date_joined (from User model), last_login (from User model)
   - Read-only serializer (source from UserProfile with user nested)

2. **UserCreateSerializer** — for POST /api/users/:
   - Input fields: username, email, first_name, last_name, role_id (integer), phone (optional), department (optional)
   - NO password field — system auto-generates a temp password
   - Validate: username uniqueness, email uniqueness, role_id exists in Role table
   - In create(): create Django User (with generated temp password), create UserProfile (role from role_id, is_force_password_change=True, is_active=True)
   - Temp password generation: 12-char random string with uppercase, lowercase, digits, and one special character (use secrets module)
   - Return: user data + temp_password field (shown once to admin)

3. **UserUpdateSerializer** — for PATCH /api/users/{id}/:
   - Writable fields: email, first_name, last_name, role_id, phone, department
   - Username is read-only after creation
   - Validate: email uniqueness (exclude current user), role_id exists

4. **UserDeactivateSerializer** — for POST /api/users/{id}/deactivate/:
   - No input fields needed (action endpoint)
   - Sets UserProfile.is_active=False AND User.is_active=False

5. **ChangePasswordSerializer** — for POST /api/users/change-password/:
   - Input fields: current_password (required), new_password (required), confirm_password (required)
   - Validate: current_password matches request.user's actual password (using check_password)
   - Validate: new_password == confirm_password
   - Validate: new_password meets minimum requirements (at least 8 chars, at least one uppercase, one lowercase, one digit)
   - Validate: new_password != current_password (must actually change)
  </action>

<task type="auto">
  <name>Task 2: User management views and URL routes</name>
  <files>backend/core/views.py, backend/core/urls.py</files>
  <action>
Add user management views to core/views.py:

1. **UserListCreateView** (APIView, permission_classes=[IsAdmin]):
   - GET: List all users with server-side pagination (page_size=20, same pattern as will be used for customers). Query UserProfile.objects.select_related('user', 'role'). Support ?search= query param (searches username, email, first_name, last_name). Support ?is_active= filter (true/false). Return paginated response: {count, next, previous, results}.
   - POST: Create user using UserCreateSerializer. On success, return 201 with user data + temp_password. Log CREATE audit entry with ip_address using get_client_ip().

2. **UserDetailView** (APIView, permission_classes=[IsAdmin]):
   - GET: Return single user by UserProfile id (or User id — use User.id as the external identifier since UserProfile is 1:1).
   - PATCH: Update user using UserUpdateSerializer. Log UPDATE audit entry.

3. **UserDeactivateView** (APIView, permission_classes=[IsAdmin]):
   - POST /api/users/{id}/deactivate/: Set User.is_active=False and UserProfile.is_active=False. Invalidate all sessions by blacklisting all outstanding refresh tokens for this user (query OutstandingToken from simplejwt and blacklist each). Log UPDATE audit entry with description "Deactivated user: {username}".

4. **UserActivateView** (APIView, permission_classes=[IsAdmin]):
   - POST /api/users/{id}/activate/: Set User.is_active=True and UserProfile.is_active=True. Log UPDATE audit entry.

5. **RoleListView** (APIView, permission_classes=[IsAdmin]):
   - GET /api/roles/: Return all Role objects [{id, name, description}]. No pagination needed (only 3 roles).

6. **ChangePasswordView** (APIView, permission_classes=[IsAuthenticated]):
   - POST /api/users/change-password/: Accepts current_password, new_password, confirm_password.
   - Uses ChangePasswordSerializer for validation (passes request in serializer context).
   - On valid: set request.user.set_password(new_password), save user. Also set request.user.userprofile.is_force_password_change=False, save profile.
   - Return 200 with {message: "Password changed successfully"}.
   - Note: This endpoint is IsAuthenticated (not IsAdmin) because any logged-in user who needs to change their password should be able to call it.

Add URL patterns to core/urls.py:
```
path("users/", UserListCreateView.as_view(), name="user_list_create"),
path("users/change-password/", ChangePasswordView.as_view(), name="change_password"),
path("users/<int:pk>/", UserDetailView.as_view(), name="user_detail"),
path("users/<int:pk>/deactivate/", UserDeactivateView.as_view(), name="user_deactivate"),
path("users/<int:pk>/activate/", UserActivateView.as_view(), name="user_activate"),
path("roles/", RoleListView.as_view(), name="role_list"),
```

IMPORTANT: The change-password path MUST come before the `<int:pk>` path, otherwise "change-password" would be matched as a pk parameter.

Import IsAdmin from core.permissions. Use get_client_ip from core.utils for audit logging. For token blacklisting on deactivation, import OutstandingToken and BlacklistedToken from rest_framework_simplejwt.token_blacklist.models.
