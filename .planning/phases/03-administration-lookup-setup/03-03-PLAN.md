**Create backend/core/audit.py** with:

1. **AuditLogMixin** — a DRF mixin class that views can inherit to auto-log CRUD:
   - Override perform_create(): after super().perform_create(), create AuditLog entry with action="CREATE", model_name=model class name, object_id=instance.pk, object_repr=str(instance), ip_address=get_client_ip(request)
   - Override perform_update(): after super().perform_update(), create AuditLog entry with action="UPDATE"
   - Override perform_destroy(): BEFORE super().perform_destroy(), capture str(instance) and pk, then after destroy create AuditLog with action="DELETE"
   - Helper method _log_action(request, action, instance, description="") that creates the AuditLog entry

2. **log_audit()** standalone function for manual audit logging (used by Plan 01's user management views and other non-ViewSet endpoints):
   - Signature: log_audit(user, action, model_name, object_id, object_repr, ip_address, changes=None)
   - Creates AuditLog.objects.create() with the provided fields

**In core/serializers.py**, add:

3. **AuditLogSerializer** — read-only serializer for audit log entries:
   - Fields: id, user (username string, not nested), action, model_name, object_id, object_repr, changes (JSON), ip_address, timestamp
   - user field: SerializerMethodField that returns user.username or "System" if null

<task type="auto">
  <name>Task 2: Audit log API endpoint with filters</name>
  <files>backend/core/views.py, backend/core/urls.py</files>
  <action>
Add to core/views.py:

**AuditLogListView** (APIView, permission_classes=[IsAdmin]):
- GET /api/audit-log/ — returns paginated audit log entries (page_size=25)
- Filter query params:
  - ?user={username} — filter by acting user's username (case-insensitive contains)
  - ?action={CREATE|UPDATE|DELETE|LOGIN|LOGOUT} — filter by action type
  - ?module={model_name} — filter by model_name (case-insensitive contains), e.g. "User", "Customer", "Job"
  - ?date_from={YYYY-MM-DD} — filter entries on or after this date
  - ?date_to={YYYY-MM-DD} — filter entries on or before this date (inclusive, so add 1 day)
  - ?search={text} — search in object_repr (case-insensitive contains)
- Ordering: newest first (default, -timestamp)
- Return format: {count, next, previous, results} — same pagination pattern as user list

Add URL pattern to core/urls.py:
```
path("audit-log/", AuditLogListView.as_view(), name="audit_log_list"),
```

NOTE: Plan 01's user management views already use inline audit logging calls. After this plan creates core/audit.py with log_audit(), those views should ideally use log_audit() instead of inline AuditLog.objects.create(). However, both approaches create the same AuditLog records — this refactor can be done as a follow-up within the same execution context or in a future plan. Do NOT modify Plan 01's views in this plan; this plan only adds the audit log API endpoint and the audit.py module.
