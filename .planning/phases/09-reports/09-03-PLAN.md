    **Expandable drilldown rows (LOCKED — user decision):**

    Each summary row is clickable and toggles an expanded state showing individual jobs for that customer. This is NOT a flat list — it is an inline drilldown sub-table beneath the summary row, on demand.

    Row expand/collapse mechanism:
    - Add a `expandedCustomerId: number | null` state (only one customer expanded at a time — clicking an already-open row collapses it, clicking a different row closes the previous and opens the new one).
    - The Customer cell (or a chevron icon prepended to it) is the click target. Show a right-pointing chevron (▶) when collapsed, down-pointing (▼) when expanded. Use Tailwind `transition-transform` for the icon rotation.
    - The summary row gets a subtle hover background to hint it is clickable (`cursor-pointer hover:bg-slate-50`).

    Drilldown data fetching:
    - When a row is expanded, fetch individual jobs from the Phase 5 jobs API (already supports these filters — no new backend endpoint needed):

    - Use `apiFetch` from `@/lib/api`. The response shape is the paginated jobs list from Phase 5 (05-03): `{ results: JobListItem[], count: number, ... }`.
    - Cache fetched drilldown data in a `drilldownCache: Record<number, JobListItem[]>` ref so re-expanding the same customer does not re-fetch.
    - Per-customer drilldown loading state: `drilldownLoading: Set<number>` — while loading, show a spinner row inside the expanded section.

    Drilldown sub-table (rendered as an indented block below the summary row, spanning all columns via `colspan`):
    - Columns: Job Number | Status | Job Type | Created Date | Cost
    - Job Number: link to `/jobs/{id}` (same navigation pattern as Phase 5 job list).
    - Status: render the status badge/pill matching the job list component from Phase 5.
    - Job Type: human-readable label (air/sea/road — match Phase 5 display).
    - Created Date: format as `DD MMM YYYY`.
    - Cost: format as `GHS {value}` using the same decimal formatter as the summary row.
    - Loading state: a single row with a centred spinner while fetching.
    - Empty state: a single row with centred grey text "No jobs found." if `results` is empty.
    - The sub-table has a slightly indented left padding and a light grey background (`bg-slate-50`) to visually distinguish it from the summary rows.
    - No pagination on the drilldown — `page_size=50` is the cap. If `count > 50`, show a note: "Showing first 50 jobs — use the Jobs page to see all."
