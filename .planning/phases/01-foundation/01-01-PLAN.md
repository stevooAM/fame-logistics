  <action>
    Create `backend/Dockerfile`:
    - Base: `python:3.12-slim`
    - Install system deps for psycopg2: `libpq-dev gcc`
    - Copy requirements.txt, pip install
    - Copy source code
    - Expose port 8000
    - CMD: `python manage.py runserver 0.0.0.0:8000`
    - Create a placeholder `backend/requirements.txt` with: django>=5.0, djangorestframework, djangorestframework-simplejwt, psycopg2-binary, celery[redis], redis, django-cors-headers, python-dotenv, ruff, pytest, pytest-django, gunicorn

    Create `frontend/Dockerfile`:
    - Base: `node:20-alpine`
    - WORKDIR /app
    - Copy package.json and package-lock.json (will exist after Plan 03)
    - RUN npm install
    - Copy source code
    - Expose port 3000
    - CMD: `npm run dev`
    - Create a placeholder `frontend/package.json` with name "fame-fms-frontend", version "0.1.0", and a dev script: `next dev`

    Create `docker-compose.yml` at project root:
    ```yaml
    services:
      db:
        image: postgres:16-alpine
        env_file: .env
        environment:
          POSTGRES_DB: ${POSTGRES_DB}
          POSTGRES_USER: ${POSTGRES_USER}
          POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
        ports:
          - "5432:5432"
        volumes:
          - postgres_data:/var/lib/postgresql/data
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
          interval: 5s
          timeout: 5s
          retries: 5

      redis:
        image: redis:7-alpine
        ports:
          - "6379:6379"
        healthcheck:
          test: ["CMD", "redis-cli", "ping"]
          interval: 5s
          timeout: 5s
          retries: 5

      backend:
        build:
          context: ./backend
          dockerfile: Dockerfile
        env_file: .env
        ports:
          - "8000:8000"
        volumes:
          - ./backend:/app
        depends_on:
          db:
            condition: service_healthy
          redis:
            condition: service_healthy
        command: python manage.py runserver 0.0.0.0:8000

      frontend:
        build:
          context: ./frontend
          dockerfile: Dockerfile
        env_file: .env
        ports:
          - "3000:3000"
        volumes:
          - ./frontend:/app
          - /app/node_modules
        depends_on:
          - backend

    volumes:
      postgres_data:
    ```

    Copy .env.example to .env so Docker can use it immediately: `cp .env.example .env`

    Note: The backend and frontend services will not fully start until Plans 02 and 03 create the actual Django/Next.js projects. This plan creates the scaffolding. The Dockerfiles use placeholder files that will be replaced.
  </action>
