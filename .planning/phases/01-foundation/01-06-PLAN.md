  <name>Task 1: GitHub Actions CI workflow with PostgreSQL service</name>
  <files>

  </files>
  <action>
    Create `.github/workflows/ci.yml`:

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    env:
      POSTGRES_DB: fame_fms_test
      POSTGRES_USER: fame_test
      POSTGRES_PASSWORD: test_password
      POSTGRES_HOST: localhost
      POSTGRES_PORT: 5432
      DJANGO_SECRET_KEY: test-secret-key
      DJANGO_DEBUG: "True"
      DJANGO_ALLOWED_HOSTS: localhost
      REDIS_URL: redis://localhost:6379/0

    jobs:
      backend-lint:
        name: Backend Lint (Ruff)
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-python@v5
            with:
              python-version: "3.12"
          - run: pip install ruff
          - run: ruff check backend/
          - run: ruff format --check backend/

      backend-test:
        name: Backend Tests (pytest)
        runs-on: ubuntu-latest
        services:
          postgres:
            image: postgres:16-alpine
            env:
              POSTGRES_DB: fame_fms_test
              POSTGRES_USER: fame_test
              POSTGRES_PASSWORD: test_password
            ports:
              - 5432:5432
            options: >-
              --health-cmd pg_isready
              --health-interval 10s
              --health-timeout 5s
              --health-retries 5
          redis:
            image: redis:7-alpine
            ports:
              - 6379:6379
            options: >-
              --health-cmd "redis-cli ping"
              --health-interval 10s
              --health-timeout 5s
              --health-retries 5
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-python@v5
            with:
              python-version: "3.12"
          - run: pip install -r backend/requirements.txt
          - run: cd backend && python manage.py migrate --no-input
          - run: cd backend && pytest -v

      frontend-lint:
        name: Frontend Lint (ESLint) + Typecheck
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: "20"
              cache: "npm"
              cache-dependency-path: frontend/package-lock.json
          - run: cd frontend && npm ci
          - run: cd frontend && npx eslint . --max-warnings 0
          - run: cd frontend && npx tsc --noEmit

      frontend-build:
        name: Frontend Build (Next.js)
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: "20"
              cache: "npm"
              cache-dependency-path: frontend/package-lock.json
          - run: cd frontend && npm ci
          - run: cd frontend && npm run build
    ```

    All four jobs run in parallel. All are required checks (configure via GitHub branch protection rules — the checkpoint below will prompt the user to set this up).

    Create `backend/ruff.toml`:
    ```toml
    line-length = 120
    target-version = "py312"

    [lint]
    select = ["E", "F", "I", "UP", "B", "SIM"]
    ignore = ["E501"]

    [format]
    quote-style = "double"
    indent-style = "space"
    ```

    Ensure `frontend/.eslintrc.json` extends `next/core-web-vitals` (should already exist from Plan 03 — verify and update if needed).

    For Vercel preview deploys: Add a comment block in the workflow file:
    ```yaml

    # Connect this repo at https://vercel.com/new to enable PR previews
    # Vercel will auto-deploy the frontend/ directory on every PR
    ```
  </action>
  <verify>
    Validate workflow syntax: `cat .github/workflows/ci.yml | python3 -c "import sys, yaml; yaml.safe_load(sys.stdin); print('Valid YAML')"`.
    Run linters locally to verify they pass:
    - `cd backend && ruff check . && ruff format --check .`
    - `cd frontend && npx eslint . --max-warnings 0`

    All should pass before committing.

Uses real PostgreSQL 16 and Redis 7 service containers. All checks pass locally. Ruff is configured for Python linting. Vercel integration instructions documented in workflow comments.</done>
