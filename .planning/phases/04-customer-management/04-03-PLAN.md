**Command behavior:**

1. **File location:** Accept an optional `--file` argument (default: look for `fame_logistic_customers.xlsx` in the project root directory, i.e., `settings.BASE_DIR / "fame_logistic_customers.xlsx"`). Also check `settings.BASE_DIR / "data" / "fame_logistic_customers.xlsx"` as a fallback.

2. **Graceful skip:** If the file does not exist at either location, print a warning: "Excel file not found at [path]. Skipping customer seed. Place the file and re-run." and exit with code 0 (NOT an error).

3. **Read Excel:** Use openpyxl to load the workbook (read_only=True for memory efficiency). Read the first sheet. The first row is headers.

4. **Column mapping:** The CONTEXT says "column names map directly to DB fields." Map Excel columns to Customer model fields. Use case-insensitive header matching. Expected columns may include variations like:
   - "Company Name" or "company_name" -> company_name
   - "TIN" or "tin" -> tin
   - "Contact Person" or "contact_person" -> contact_person
   - "Phone" or "phone" -> phone
   - "Email" or "email" -> email
   - "Address" or "address" -> address
   - "Customer Type" or "customer_type" or "Type" -> customer_type
   - "Business Type" or "business_type" -> business_type
   - "Credit Terms" or "credit_terms" -> credit_terms
   - "Notes" or "notes" -> notes

   Create a COLUMN_MAP dict that normalizes headers. Skip unrecognized columns with a warning.

5. **Validation per row:**
   - company_name is required — skip row if missing/empty (log warning with row number)
   - Trim whitespace from all string fields
   - customer_type: if provided and not "Company" or "Individual", default to "Company"
   - All other fields: treat empty/None as blank string

6. **Idempotency:** Use `Customer.objects.update_or_create(tin=row_tin, defaults={...})` when TIN is non-empty. If TIN is empty, use `Customer.objects.get_or_create(company_name=row_company_name, defaults={...})`. This prevents duplicates on re-run.

7. **Output:** Print summary: "Seeded {created} new customers, updated {updated} existing, skipped {skipped} invalid rows (total: {total} rows processed)."

8. **Do NOT** try to resolve preferred_port or currency_preference FKs from the Excel — those fields are for manual entry in the new system. Only seed the flat text fields that exist in the Excel.

9. **Transaction:** Wrap the entire import in `transaction.atomic()` so it's all-or-nothing.
