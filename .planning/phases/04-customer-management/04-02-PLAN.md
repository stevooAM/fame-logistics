**backend/customers/views.py:**

Create CustomerViewSet inheriting from AuditLogMixin and ModelViewSet:

1. **Queryset:** Customer.objects.select_related("preferred_port", "currency_preference").filter(is_active=True) by default. Support `?include_inactive=true` query param to include soft-deleted records.

2. **Permissions:** permission_classes = [IsAdminOrOperations]

3. **Pagination:** Use DRF PageNumberPagination. Create a CustomerPagination class:
   - page_size = 20
   - page_size_query_param = "page_size"
   - max_page_size = 100

4. **Serializer:** Use CustomerListSerializer for list action, CustomerSerializer for all other actions. Override get_serializer_class().

5. **Filtering:** Override get_queryset() to support these query params:
   - search: icontains on company_name, tin, contact_person, email (OR logic)
   - company_name: icontains filter
   - tin: icontains filter
   - business_type: icontains filter
   - customer_type: exact filter
   - credit_terms: icontains filter
   - ordering: order_by field (default: company_name). Support: company_name, -company_name, tin, -tin, created_at, -created_at

6. **Soft-delete:** Override destroy() method — set is_active=False, save, log audit via _log_action, return Response with serialized data and status 200 (matching the pattern from setup/views.py LookupViewSetMixin).

7. **Duplicate TIN check endpoint:** Add a custom action:
   ```python
   @action(detail=False, methods=["get"], url_path="check-tin")
   def check_tin(self, request):
   ```
   Accepts `?tin=XXX&exclude_id=YY` query params.
   Returns: {"duplicate": bool, "existing_customer": {"id": N, "company_name": "..."} | null}
   Searches all customers (including inactive) for matching TIN. If exclude_id is provided, excludes that customer from the check (for edit mode).
