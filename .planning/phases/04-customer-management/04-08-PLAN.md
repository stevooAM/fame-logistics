<task type="auto">
  <name>Task 1: Create entrypoint.sh and wire into Dockerfile</name>
  <files>
    backend/entrypoint.sh
    backend/Dockerfile
  </files>
  <action>
    Create `backend/entrypoint.sh` with the following content exactly:

    ```sh
    #!/bin/sh
    set -e

    echo "Running database migrations..."
    python manage.py migrate --no-input

    echo "Seeding customers (skips gracefully if Excel file absent)..."
    python manage.py seed_customers

    echo "Starting development server..."
    exec python manage.py runserver 0.0.0.0:8000
    ```

    Key points:
    - `set -e` causes the script to abort on any non-zero exit — but seed_customers already exits 0 when file is missing, so a missing Excel file does NOT abort startup
    - `exec` replaces the shell process with runserver so PID 1 signals (SIGTERM) reach Django correctly
    - Use `#!/bin/sh` not `#!/bin/bash` — the python:3.12-slim base image has sh, not bash

    Then update `backend/Dockerfile` to:
    1. After the existing `COPY . .` line, add:
       ```dockerfile
       COPY entrypoint.sh /entrypoint.sh
       RUN chmod +x /entrypoint.sh
       ```
    2. Replace the existing `CMD ["python", "manage.py", "runserver", "0.0.0.0:8000"]` with:
       ```dockerfile
       ENTRYPOINT ["/entrypoint.sh"]
       ```

    The final Dockerfile should read:
    ```dockerfile
    FROM python:3.12-slim

    WORKDIR /app

    RUN apt-get update && apt-get install -y --no-install-recommends \
        libpq-dev \
        gcc \
        && rm -rf /var/lib/apt/lists/*

    COPY requirements.txt .
    RUN pip install --no-cache-dir -r requirements.txt

    COPY . .

    COPY entrypoint.sh /entrypoint.sh
    RUN chmod +x /entrypoint.sh

    EXPOSE 8000

    ENTRYPOINT ["/entrypoint.sh"]
    ```
  </action>
  <verify>
    Confirm both files exist and are syntactically correct:
    - `head -5 backend/entrypoint.sh` shows `#!/bin/sh` on line 1
    - `grep ENTRYPOINT backend/Dockerfile` returns `ENTRYPOINT ["/entrypoint.sh"]`
    - `grep -v CMD backend/Dockerfile` confirms the old CMD line is gone
    - `grep chmod backend/Dockerfile` confirms the chmod line is present
  </verify>
  <done>
    backend/entrypoint.sh exists with set -e, migrate, seed_customers, and exec runserver.
    backend/Dockerfile uses ENTRYPOINT ["/entrypoint.sh"] with no CMD override.
  </done>

    ```markdown
    # Customer Seed Data

    Place `fame_logistic_customers.xlsx` in this directory before the first deployment.

    On container startup, `entrypoint.sh` runs:

        python manage.py seed_customers

    The command auto-detects the file at `backend/data/fame_logistic_customers.xlsx`.
    If the file is absent, the command prints a warning and exits 0 — the server still starts.

    To seed manually inside a running container:

        docker compose exec backend python manage.py seed_customers
        # or with an explicit path:
        docker compose exec backend python manage.py seed_customers --file /path/to/file.xlsx
    ```
