**TinWarningDialog.tsx:**

**CustomerFormDialog.tsx:**
- shadcn/ui Dialog component
- Props: open, onClose, customer (Customer | null for create vs edit mode), onSuccess (callback after save)
- Uses React Hook Form with Zod schema validation

**Zod schema (customerFormSchema):**
- company_name: z.string().min(1, "Company name is required").max(200)
- customer_type: z.enum(["Company", "Individual"])
- tin: z.string().min(1, "TIN is required").max(50)
  - TIN is REQUIRED per user decision. Use z.string().min(1, "TIN is required").max(50).
  - Custom refinement: must match pattern for TIN prefixes (C, P, G, Q, GHA, EUROPE followed by digits/alphanumeric). Use regex: /^(C|P|G|Q|GHA|EUROPE)[\w\d-]*$/i -- but keep it lenient since the CONTEXT says "supports prefix formats" not "strictly validates"
- contact_person: z.string().max(150).optional().or(z.literal(""))
- phone: z.string().max(50).optional().or(z.literal(""))
  - Custom refinement: if non-empty, must match local Ghana formats (02x, 05x, 0xx with 9-10 digits) or international formats (+country code with digits). Regex: /^(\+?\d{1,4}[-\s]?)?\d{6,14}$/ -- lenient to accept mixed formats per CUST-09
- email: z.string().email("Invalid email").optional().or(z.literal(""))
- address: z.string().optional().or(z.literal(""))
- business_type: z.string().max(50).optional().or(z.literal(""))
- preferred_port: z.number().nullable().optional()
- currency_preference: z.number().nullable().optional()
- credit_terms: z.string().max(50).optional().or(z.literal(""))
- notes: z.string().optional().or(z.literal(""))

**Form layout:**
- Two-column grid for fields (responsive: 1 col on mobile, 2 on desktop)
- Row 1: Company Name (full width)
- Row 2: Customer Type (select), TIN
- Row 3: Contact Person, Phone
- Row 4: Email (full width)
- Row 5: Address (full width, textarea)
- Row 6: Business Type, Credit Terms
- Row 7: Preferred Port (select from /api/setup/dropdowns/), Currency Preference (select from same endpoint)
- Row 8: Notes (full width, textarea)
- Footer: Cancel + Save buttons

**Port and Currency dropdowns:** Fetch from GET /api/setup/dropdowns/ on dialog open. Use the ports and currencies arrays to populate select options.

**TIN duplicate check flow:**
1. On form submit, if TIN is non-empty, call GET /api/customers/check-tin/?tin={value}&exclude_id={customer.id if editing}
2. If duplicate=true, open TinWarningDialog with the existing customer name
3. If user clicks "Proceed Anyway", continue with the save
4. If user clicks "Cancel", return to the form

**Save flow:**
- Create: POST /api/customers/ with form data via apiFetch
- Edit: PATCH /api/customers/{id}/ with form data via apiFetch
- On success: call onSuccess callback, close dialog
- On error: show field-level errors from API response (DRF returns {field: [errors]})

All API calls use apiFetch<T>.
