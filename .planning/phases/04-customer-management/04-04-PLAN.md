**BatchActionBar.tsx (CUST-06 -- batch save/cancel):**
- A sticky bar that appears ABOVE the grid (or below the toolbar) when there are unsaved changes
- Shows count of modified rows: "{N} unsaved change(s)"
- Two buttons: "Save All" (primary teal) and "Cancel" (secondary/outline)
- "Save All" calls parent callback which iterates dirty rows and PATCHes each via apiFetch
- "Cancel" calls parent callback which reverts all dirty rows to their original values
- Accept props: dirtyCount: number, onSaveAll: () => void, onCancelAll: () => void, isSaving: boolean
- When isSaving is true, show a spinner on the Save All button and disable both buttons

**CustomerTable.tsx:**
- AG Grid React component with client-side model and manual pagination (AG Grid Community)
- **Columns** (per CONTEXT.md locked decision):
  - Company Name (sortable, filterable, editable)

  - Contact Person (sortable, editable)

  - Business Type (sortable, filterable, editable)
  - Credit Terms (sortable, editable)
  - Actions column: kebab menu (three dot icon) with dropdown containing Edit (modal), View Detail, Delete
- **Data fetching:** Use apiFetch<CustomerListResponse> to call GET /api/customers/ with query params built from current filters, page, page_size, ordering
- **Pagination:** Custom pagination bar below the grid showing: "Showing X-Y of Z" text, page size selector (10, 20, 50), prev/next/first/last page buttons

- **Filtering:** When AG Grid column filter changes, convert to API query params and re-fetch. Use AG Grid's onFilterChanged callback.
- **Loading state:** Show loading overlay while fetching
- **Empty state:** Show "No customers found" message when results are empty

- **Inline cell editing (CUST-04) with batch tracking (CUST-06):**
  - Enable AG Grid's `editable: true` on the columns listed above
  - Do NOT save immediately on cell change. Instead, track dirty rows in component state:
    - Maintain a `dirtyRows` Map<number, Partial<Customer>> keyed by row id
    - On `onCellValueChanged`, add/update the row id in dirtyRows with the changed fields
    - Visually mark dirty rows (e.g., light yellow background or a small dot indicator on the row)
  - Expose dirtyRows count and save/cancel callbacks to parent via props or a ref
  - **Save All (CUST-06):** For each entry in dirtyRows, call PATCH /api/customers/{id}/ with changed fields. On success, clear that entry from dirtyRows. On error, keep it dirty and show error toast. After all saves complete, re-fetch the grid data.
  - **Cancel (CUST-06):** Revert all dirty cells to their original fetched values. Clear dirtyRows. Use AG Grid's `api.undoCellEditing()` if available, or re-fetch the data to reset.

- **Inline row creation (CUST-03 -- blank row at top of grid):**
  - When "Add Inline" is triggered, insert a temporary empty row at index 0 of the grid data with a temporary negative id (e.g., -Date.now())
  - This row is editable inline just like existing rows
  - Track new rows separately from dirty existing rows (newRows Set or similar)
  - On "Save All", new rows call POST /api/customers/ instead of PATCH
  - On "Cancel", remove the temporary row from grid data
  - Validation: company_name is required before save -- show cell-level error styling if empty on save attempt

- **Row actions:** Kebab menu renders a dropdown. Edit triggers parent callback (modal), View Detail navigates to /customers/[id], Delete triggers parent callback.
- Accept props: onEditCustomer, onDeleteCustomer, refreshTrigger (number that increments to force re-fetch), filters
