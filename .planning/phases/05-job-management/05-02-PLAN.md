<task type="auto">
  <name>Task 2: Create cloud storage utility module</name>
  <files>backend/jobs/storage.py</files>
  <action>
Create `backend/jobs/storage.py` with three functions:

1. `_get_s3_client()` — internal helper that returns a boto3 S3 client configured with settings.AWS_* values. Uses `boto3.client("s3", endpoint_url=..., aws_access_key_id=..., aws_secret_access_key=..., region_name=...)`. Cache the client in a module-level variable for reuse.

2. `upload_document(file_obj, key: str) -> str`:
   - Accepts a Django UploadedFile (or any file-like object) and a storage key (e.g., `jobs/{job_id}/documents/{uuid}_{filename}`)
   - Uploads to the configured bucket using `client.upload_fileobj(file_obj, bucket, key)`
   - Returns the full storage key (not a URL — presigned URLs are generated separately)
   - Raises a custom `StorageError` exception on failure (wraps botocore exceptions)

3. `delete_document(key: str) -> None`:
   - Deletes the object at the given key from the bucket
   - Silently succeeds if the key doesn't exist (idempotent)
   - Logs errors but does not raise (deletion failure should not break the app)

4. `get_presigned_url(key: str, expiry: int | None = None) -> str`:
   - Generates a presigned GET URL for the given key
   - Uses `settings.AWS_PRESIGNED_URL_EXPIRY` as default expiry (1 hour)
   - Returns the presigned URL string

5. `StorageError(Exception)` — custom exception class for upload failures.

Add a module docstring explaining the R2/S3 compatibility and that credentials must be configured via environment variables. Include a guard that logs a warning if AWS_ACCESS_KEY_ID is empty (storage operations will fail gracefully).

Import pattern: `from django.conf import settings` for all settings access. `import logging` for the logger.
