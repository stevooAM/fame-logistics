<task type="auto">
  <name>Task 2: Create Job ViewSet with all endpoints</name>
  <files>backend/jobs/views.py, backend/jobs/urls.py, backend/config/urls.py</files>
  <action>
Create `backend/jobs/views.py` with a JobViewSet following the CustomerViewSet pattern:

**JobPagination**: page_size=20, page_size_query_param="page_size", max_page_size=100.

**JobViewSet** (AuditLogMixin, ModelViewSet):
- permission_classes: [IsAdminOrOperations] (Finance is view-only — handled separately)
- pagination_class: JobPagination

**get_permissions()** override:
- For "list", "retrieve", "audit_trail", "documents": use IsAnyRole (Finance can view)
- For all write actions: use IsAdminOrOperations

**get_queryset():**
- select_related("customer", "assigned_to", "created_by")
- Default ordering: `-created_at` (newest first per CONTEXT)
- Filters (all from query params, defensive — malformed params silently ignored per project pattern):
  - `search`: OR across job_number, customer__company_name, cargo_description, bill_of_lading, container_number (icontains)
  - `status`: exact match on JobStatus value
  - `job_type`: exact match on JobType value
  - `customer_id`: exact FK match
  - `customer_name`: icontains on customer__company_name
  - `assigned_to`: exact FK match
  - `date_from` / `date_to`: filter on created_at (date_to adds timedelta(days=1) with __lt for inclusive end — pattern from 03-03)
  - `port`: icontains on origin or destination
  - `ordering`: allow job_number, -job_number, created_at, -created_at, status, -status, customer__company_name, -customer__company_name, total_cost, -total_cost

**get_serializer_class():**
- list action: JobListSerializer
- Otherwise: JobSerializer

**perform_create() override:**
- Set `serializer.save(created_by=request.user)` — job_number auto-assigned by model save()
- Create JobAuditTrail entry: action="CREATED", new_value=job.status

**Custom actions:**

1. `@action(detail=True, methods=["patch"], url_path="transition")` — **transition_status**:
   - Accepts StatusTransitionSerializer data (new_status, optional reason)
   - **Status flow rules** (LOCKED decisions):
     - Define ALLOWED_TRANSITIONS dict mapping current_status -> list of allowed next statuses
     - Forward transitions: DRAFT->PENDING, PENDING->IN_PROGRESS, IN_PROGRESS->CUSTOMS, CUSTOMS->DELIVERED, DELIVERED->CLOSED
     - CANCELLED: any status except CLOSED can go to CANCELLED
     - PENDING status also allows "Submit for Approval" — this is the same PENDING status, it creates an approval queue entry (stub: create a record in approvals.ApprovalQueue if model exists, otherwise just log it — Phase 6 will wire this)
     - **Reversal**: Any backward transition is Admin-only. Check user role — if Operations and the transition goes backward, return 403 with message "Only Admin can reverse job status"
   - On valid transition:
     - Save old_status, update job.status to new_status, save job
     - Create JobAuditTrail entry: action="STATUS_CHANGE", old_value=old_status, new_value=new_status, user=request.user
     - If new_status is PENDING and this is a "submit for approval" scenario, also create JobAuditTrail entry with action="SUBMITTED_FOR_APPROVAL"
     - Return updated job via JobSerializer
   - On invalid transition: return 400 with error message listing allowed transitions

2. `@action(detail=True, methods=["get"], url_path="audit-trail")` — **audit_trail**:
   - Returns paginated list of JobAuditTrail entries for this job
   - Uses JobAuditTrailSerializer
   - Permission: IsAnyRole (all roles can view audit trail)

3. `@action(detail=True, methods=["post"], url_path="documents/upload")` — **upload_document**:
   - Accepts multipart/form-data with `file` (the file) and `document_type` (PK of DocumentType from setup)
   - Validates: max 20 documents per job (LOCKED decision). If at limit, return 400 "Maximum 20 documents per job"
   - Validates: document_type is required (LOCKED decision)
   - Generates storage key: `jobs/{job.id}/documents/{uuid4}_{original_filename}`
   - Calls `storage.upload_document(file, key)`
   - Creates JobDocument record (file_name=original name, file_url=key, file_size=file.size, document_type=type, uploaded_by=request.user)
   - Creates JobAuditTrail entry: action="DOCUMENT_UPLOADED", new_value=filename
   - Returns JobDocumentSerializer data, status 201
   - On StorageError: return 500 with "Document upload failed"

4. `@action(detail=True, methods=["get"], url_path="documents")` — **list_documents**:
   - Returns all JobDocument entries for this job via JobDocumentSerializer
   - Each document's file_url is replaced with a presigned URL via `storage.get_presigned_url(doc.file_url)`
   - Permission: IsAnyRole

5. `@action(detail=True, methods=["delete"], url_path="documents/(?P<doc_id>[0-9]+)")` — **delete_document**:
   - Deletes a specific JobDocument by doc_id
   - Calls `storage.delete_document(doc.file_url)` to remove from cloud storage
   - Creates JobAuditTrail entry: action="DOCUMENT_DELETED", old_value=filename
   - Returns 200 with success message
   - Permission: IsAdminOrOperations

**destroy() override (soft-delete pattern):**
- Do NOT allow deletion of jobs — return 405 "Job deletion is not permitted. Use status CANCELLED instead."

**URL configuration:**

- Register JobViewSet at "" with basename="job"
- Update `backend/config/urls.py` to add `path("api/jobs/", include("jobs.urls"))`
