<task type="auto">
  <name>Task 2: Create JobFormDialog with all fields and validation</name>
  <files>frontend/src/app/(dashboard)/jobs/components/JobFormDialog.tsx</files>
  <action>
Create JobFormDialog following the CustomerFormDialog pattern (React Hook Form + Zod + Dialog):

Props:
- `open: boolean`
- `onClose: () => void`
- `job?: Job | null` — if provided, edit mode; if null/undefined, create mode
- `onSuccess: () => void` — called after successful save

**Zod schema** (jobFormSchema):
- customer: z.number({ required_error: "Customer is required" }).min(1, "Customer is required")
- job_type: z.enum(["IMPORT", "EXPORT", "TRANSIT", "LOCAL"], { required_error: "Job type is required" })
- origin: z.string().max(200).optional().default("")
- destination: z.string().max(200).optional().default("")
- cargo_description: z.string().min(1, "Cargo description is required")
- bill_of_lading: z.string().min(1, "Bill of Lading is required").max(100)
- container_number: z.string().min(1, "Container number is required").max(50)
- weight_kg: z.string().min(1, "Weight is required").refine(val => !isNaN(parseFloat(val)) && parseFloat(val) > 0, "Must be a positive number")
- volume_cbm: z.string().min(1, "Volume is required").refine(val => !isNaN(parseFloat(val)) && parseFloat(val) > 0, "Must be a positive number")
- total_cost: z.string().min(1, "Cost is required").refine(val => !isNaN(parseFloat(val)) && parseFloat(val) >= 0, "Must be a non-negative number")
- notes: z.string().optional().default("")
- eta: z.string().optional().default("")
- delivery_date: z.string().optional().default("")
- assigned_to: z.number().nullable().optional()

**Form layout** using FieldGroup sections (established pattern from 04-05):
- Section "Job Information": job_type (select dropdown), customer (CustomerPicker), assigned_to (staff picker — simple select fetching from /api/users/ or text input)
- Section "Shipment Details": origin, destination, cargo_description (textarea), bill_of_lading, container_number
- Section "Measurements & Cost": weight_kg, volume_cbm, total_cost
- Section "Dates": eta (date input), delivery_date (date input)
- Section "Notes": notes (textarea, optional)

**Submit behavior:**
- Create mode: POST /api/jobs/ with form data (exclude job_number — auto-generated)
- Edit mode: PATCH /api/jobs/{job.id}/ with changed fields
- On success: call onSuccess, show no toast (consistent with customer form pattern)
- On API error: display field-level errors from DRF response (same pattern as CustomerFormDialog)
- Loading state on submit button ("Creating..." / "Saving...")

**Dialog structure:**
- Title: "Create New Job" or "Edit Job {job.job_number}"
- Description: "Fill in the shipment details below"
- Cancel and Submit buttons at bottom
- Use Dialog, DialogContent, DialogHeader, DialogTitle, DialogDescription from @/components/ui/dialog
- Use Button from @/components/ui/button
- Use inputStyle() helper for all input fields
- ScrollArea or max-height with overflow-y-auto for the form body (many fields)

**Edit mode prefill:**
- When job prop is provided, reset form with job values on open
- In edit mode, customer picker shows the linked customer
- job_number is displayed as read-only text (not an editable field)
